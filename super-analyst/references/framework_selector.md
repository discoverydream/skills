
## 问题维度的智能识别

系统首先通过多层扫描对问题进行分类：

### 复杂度评估层级
| 级别 | 特征描述 | 典型问题示例 | 建议时间 |
| --- | --- | --- | --- |
| Level 1 | 简单事实查询、概念解释 | "什么是金字塔原理？" | 1-2分钟 |
| Level 2 | 需部分信息、2-3维度分析 | "如何用费曼学习法提升学习效率？" | 3-6分钟 |
| Level 3 | 深度研究、多维度、战略决策 | "企业数字化转型战略规划" | 10-20分钟 |

### 时间维度分类
| 维度类型 | 特征 | 适用框架 |
| --- | --- | --- |
| 现状描述 | 当前状态分析、问题诊断 | 5W1H、MECE、冰山模型 |
| 趋势预测 | 未来走向判断、机会识别 | 情景规划、AIPL模型、成长飞轮 |
| 回溯分析 | 历史原因追溯、经验总结 | 5Whys、GRAI复盘法、STAR复盘法 |

### 确定性程度分类
| 确定性类型 | 特征 | 适用框架 |
| --- | --- | --- |
| 明确答案 | 事实性问题、标准流程 | 5W1H、SMART目标原则 |
| 概率判断 | 风险评估、决策支持 | 波士顿矩阵、波特五力、成本效益分析 |
| 愿景构建 | 创新设计、变革规划 | 第一性原理、U型思考模型、设计思维 |

## 双重顺序思考的整合机制

### 第一层思考：框架适配性判断

```
输入问题 → 问题类型识别 → 复杂度评估 → 框架候选池生成
    ↓
评估维度：
├── 问题本质属性（诊断/战略/创新/决策/理解）
├── 信息完整性（完整/部分/缺失）
├── 时间跨度（即时/短期/中期/长期）
└── 利益相关者范围（个人/团队/组织/市场）
```

### 第二层思考：执行路径优化

```
框架候选池 → 组合策略设计 → 执行深度规划 → 输出形式适配
    ↓
优化维度：
├── 信息收集策略（内部/外部/混合）
├── 分析深度调整（浅层/中层/深层）
├── 时间资源配置（紧急/常规/深度）
└── 输出形式选择（简报/报告/方案）
```

## 自适应复杂度管理的精髓

### 分层处理机制
| 问题层级 | 处理策略 | 框架数量 | 输出深度 |
| --- | --- | --- | --- |
| 单点问题 | 直接框架匹配 | 1个 | 核心要点 |
| 线性问题 | 串联框架组合 | 2-3个 | 结构化分析 |
| 面状问题 | 并联框架整合 | 3-5个 | 多维洞察 |
| 系统问题 | 网络框架协同 | 5个以上 | 系统方案 |

### 迭代深化机制
```
第一轮：快速扫描 → 初步判断 → 框架初选
    ↓
第二轮：深度分析 → 信息补充 → 框架优化
    ↓
第三轮：整合验证 → 结论提炼 → 方案输出
```

### 跨界融合机制
| 融合类型 | 典型组合 | 应用场景 |
| --- | --- | --- |
| 思考+执行 | 第一性原理 + OKR工作法 | 创新项目落地 |
| 规划+复盘 | 商业模式画布 + GRAI复盘法 | 战略迭代优化 |
| 领导+情绪 | 情景领导力模型 + 情绪ABC理论 | 团队情绪管理 |
| 学习+复盘 | 费曼学习法 + KPT复盘法 | 学习效果提升 |
| 营销+沟通 | AIPL模型 + SCQA沟通模型 | 品牌故事传播 |

## 框架选择决策树

```
问题输入
    │
    ├─ 是否涉及情绪/心理？
    │   ├─ 是 → 情绪力框架（情绪ABC、ETA脱困四问等）
    │   └─ 否 ↓
    │
    ├─ 是否需要外部信息？
    │   ├─ 是 → 触发情报搜集 → 营销力/规划力框架
    │   └─ 否 ↓
    │
    ├─ 问题时间指向？
    │   ├─ 过去 → 复盘力框架（GRAI、STAR、KPT等）
    │   ├─ 现在 → 思考力/执行力框架
    │   └─ 未来 → 规划力框架（商业模式画布、情景规划等）
    │
    ├─ 涉及对象范围？
    │   ├─ 个人 → 学习力/自识力框架
    │   ├─ 团队 → 领导力/沟通力框架
    │   └─ 市场 → 营销力框架
    │
    └─ 问题复杂度？
        ├─ 简单 → 单一框架
        ├─ 中等 → 2-3框架组合
        └─ 复杂 → 多框架协同 + 迭代深化
```

---

这种选择逻辑的强大之处在于，它不是机械的框架匹配，而是基于对问题本质的深刻理解，实现分析工具的智能化组合和应用。
